Overview
--------
TinyBox is a self-contained in-browser slicer for TinyMaker printers. It:
- Loads STL models in the browser,
- Slices into per-layer PNG masks at 320×240 (default TinyMaker display),
- Observes firmware quirk: if device Layer_Height > 0.06 mm the exporter uses half-step slices so firmware's odd-indexing still produces the desired device layer height,
- Produces job.zip with a top-level folder (jobName/) containing 1.png..N.png and a manifest.json.

Quick Start
-----------
1. Clone the repo into a directory.
2. Install dependencies: `npm install`
3. Start the dev server: `npm run dev`
4. Open the local Vite URL, upload an STL or load the sample, set profile options, click "Slice", then click "Download Zip".

Notes & limits
--------------
- Default profile: 320×240 pixels. You can change pixel dims in the UI.
- Layer limit: 1080 (firmware limit). The app warns and stops if exceeded.
- For complex models with non-manifold geometry, results may need polygon cleanup; adding clipper-wasm later improves robustness.
- All slicing runs client-side — no files leave your machine.

Files
-----
- index.html: Vite mount point.
- src/App.svelte: TinyBox UI and slicing/download state.
- src/main.js: Svelte entry point.
- src/slicer-worker.js: Web Worker performing slicing and rasterization.
- manifest.json (generated): per-job parameters placed inside zip.
